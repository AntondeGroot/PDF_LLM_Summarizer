You will receive learning concepts extracted from textbook pages.

Create high-quality question cards.

Each concept may produce **at most one flashcard**.

A good flashcard should help someone:

• understand how the topic works
• reason about system behavior
• remember useful operational commands

QUESTION STYLE

Prefer questions starting with:

• Why
• How
• When
• What problem does
• How can you verify
• What command should you use to

COMMAND RULES

Commands are allowed only when they represent a **real operational task**.

Good examples:

• verifying installation
• starting a service
• troubleshooting errors
• checking status

Bad examples:

• asking what a command does
• asking about command flags
• syntax trivia

OUTPUT FORMAT (STRICT)

Output ONLY markdown.

For each page output:

===PAGE 1===
(card(s) for this page)
===END PAGE===

CARD FORMAT (STRICT)

Each card must follow this EXACT three-line structure:

Line 1: the question (may span multiple lines)
Line 2: a line with ONLY the character ? — nothing else on that line
Line 3: the answer

CRITICAL: The `?` separator is ALWAYS on its own dedicated line.
The question line ends with its natural punctuation (usually `?`), then the NEXT line is a standalone `?`.

WRONG (do not do this):
How can you verify that Docker was installed correctly?
Run ==docker run hello-world== to start a test container.

RIGHT:
How can you verify that Docker was installed correctly?
?
Run ==docker run hello-world== to start a test container that prints a confirmation message.

Separate multiple cards with a line containing exactly:

---

Answer rules:

• answers should usually be one sentence
• rewrite concepts in your own words
• highlight commands using == ==

Important:

• prefer fewer high-quality cards
• it is valid to output zero cards for a page
