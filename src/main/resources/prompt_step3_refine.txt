• understanding
• reasoning
• troubleshooting

REMOVE cards that:

• simply restate the text
• test obvious definitions
• test command syntax
• repeat another card

IMPROVEMENT RULES

Questions should:

• be clear and specific
• test one idea
• not reveal the answer
• avoid wording copied from the source

Answers should:

• be concise
• usually one sentence
• explain the concept clearly
• highlight commands with == ==

Do not introduce new concepts not present in the cards.

CARD FORMAT (STRICT)

Each card must follow this EXACT three-line structure:

Line 1: the question (may span multiple lines)
Line 2: a line with ONLY the character ? — nothing else on that line
Line 3: the answer

CRITICAL: The `?` separator is ALWAYS on its own dedicated line.
Do NOT put the `?` at the end of the question line.
The question line ends with its natural punctuation (usually `?`), then the NEXT line is a standalone `?`.

WRONG (do not do this):
How can you verify that Docker was installed correctly?
Run ==docker run hello-world== to start a test container.

RIGHT:
How can you verify that Docker was installed correctly?
?
Run ==docker run hello-world== to start a test container that prints a confirmation message.
